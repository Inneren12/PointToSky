# Screenshots — правила и чек-лист

## Телефоны/планшеты/Chromebook
- 2–8 штук. Рекомендую 6: Aim, Identify, Map, Tonight Card, AR, Settings.
- Размер: ≥1080 по короткой стороне (1080×1920 или 1920×1080). Формат: PNG/JPG.
- Убрать уведомления/индикаторы (Wi-Fi/батарея — полные).
- Первые 3 без текстовых плашек; допускаются лёгкие подписи ≤20% кадра.

## Wear OS
- 1:1 (например, 640×640), ≥384×384, только чистый UI без рамок и внешнего текста.
- Рекомендованный набор: Tile (Tonight), Identify, Card (детали объекта), Map.

## Имена файлов
`phone_01_aim.png`, `phone_02_identify.png`, `wear_01_tile.png`, …

## Съёмка
- Реальное устройство или эмулятор. Для Wear — без рамок.
- ADB пример (скрин): `adb exec-out screencap -p > phone_01_aim.png`

## Проверка перед загрузкой
- Соотношение сторон, размер, отсутствие посторонних элементов.
- Переводы (если на скриншотах есть текст).
